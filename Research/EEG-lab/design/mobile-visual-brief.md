# Mobile Visual Brief

## Mockup Goal

Generate a portrait mobile UI concept for EEG Lab that demonstrates negative space, calm onboarding, dark/light mode sensibility, and the "one idea at a time" lesson flow.

## Prompt

Use case: ui-mockup
Asset type: mobile app design concept
Primary request: a minimal mobile web app for learning EEG interpretation, with large negative space and centered guidance text.
Subject: EEG Lab onboarding and first exercise flow.
Style/medium: polished mobile UI mockup, product design concept, not a marketing poster.
Composition/framing: portrait phone screen, 390 by 844 style viewport, one main screen visible with subtle hints of a second state behind or below; lots of empty space.
Lighting/mood: calm, clinical, quiet, contemplative.
Color palette: light mode with warm white background, soft black text, muted teal signal accent; include a small dark mode thumbnail or paired mini-state if possible.
UI elements: centered title, one-sentence guidance, one primary button, tiny progress indicator, sparse EEG waveform line near lower third, minimal report excerpt panel partly revealed.
Text: use short readable UI text only, such as "EEG Lab", "Read one signal at a time.", "Begin", "Exercise 01".
Constraints: mobile-first; no dense dashboard; no busy charts; no medical stock photos; no doctors; no hero illustration; no neon; no heavy gradients; no crowded labels; no fake patient identifiers.
Avoid: clutter, hospital-blue dashboard, oversized cards, nested cards, decorative blobs, purple-blue gradient hero, illegible tiny paragraphs.

## How To Use The Mockup

Use the mockup as visual intent only. The implemented app should be built with HTML/CSS and the existing EEG canvas, not by embedding the mockup image as UI.
